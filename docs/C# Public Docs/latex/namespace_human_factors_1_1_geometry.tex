\doxysubsection{Detailed Description}
Manipulate and load geometry from disk. 

The \mbox{\hyperlink{namespace_human_factors_1_1_geometry}{Geometry}} namespace contains all functionality in \mbox{\hyperlink{namespace_human_factors}{Human\+Factors}} for dealing with mesh \mbox{\hyperlink{namespace_human_factors_1_1_geometry}{Geometry}}. The fundamental datatype of \mbox{\hyperlink{namespace_human_factors_1_1_geometry}{Human\+Factors.\+Geometry}} is \mbox{\hyperlink{class_human_factors_1_1_geometry_1_1_mesh_info}{Mesh\+Info}}, which stores the vertices and indices of a mesh in unmanaged memory.

\begin{DoxyRemark}{Remarks}
Right now only O\+BJ files are supported for loading geometry from disk, however \mbox{\hyperlink{class_human_factors_1_1_geometry_1_1_mesh_info}{Mesh\+Info}} offers multiple constructors that accept arrays of vertices and indices so support for more formats can be developed.
\end{DoxyRemark}
