\doxysubsection{Detailed Description}
Algorithms to find the shortest path between nodes in a \mbox{\hyperlink{a01199}{H\+F\+::\+Spatial\+Structures\+::\+Graph}}. 

This namespace utilizes pathfinding algorithms from Boost internally.\+To make use of the algorithms in this namespace, first you must create a \mbox{\hyperlink{a01139}{Boost\+Graph}} from the \mbox{\hyperlink{a01199}{H\+F\+::\+Spatial\+Structures\+::\+Graph}} you want to find paths in. ~\newline

\mbox{\hyperlink{a01215}{H\+F\+::\+Spatial\+Structures\+::\+Path}} for a description of the path datatype.

To find the path, A row of the predecessor matrix is generated for node a, then followed until node B is reached. This algorithm is implemented using dijkstra\+\_\+shortest\+\_\+path from the Boost\+Graph\+Library \href{https://www.boost.org/doc/libs/1_73_0/libs/graph/doc/dijkstra_shortest_paths_no_color_map.html}{\texttt{ https\+://www.\+boost.\+org/doc/libs/1\+\_\+73\+\_\+0/libs/graph/doc/dijkstra\+\_\+shortest\+\_\+paths\+\_\+no\+\_\+color\+\_\+map.\+html}}

\begin{DoxyNote}{Note}
Use Find\+Paths for multiple paths as it\textquotesingle{}s able to reuse a lot of work compared to running this in a loop.
\end{DoxyNote}

\begin{DoxyRemark}{Remarks}

It\textquotesingle{}s efficent and safe for that purpose, but \mbox{\hyperlink{a00282_a15badb9d0a17849de7b21105605c9c91}{Find\+Paths}} should be preferred outside of that context since this function can be quite dangerous if not handled properly.
\end{DoxyRemark}
