\doxysubsection{Detailed Description}
Manipulate and load geometry from disk. 

\mbox{\hyperlink{a00312}{H\+F\+::\+Geometry}} contains all functionality in Human\+Factors for dealing with mesh \mbox{\hyperlink{a00312}{Geometry}}. The fundamental datatype of \mbox{\hyperlink{a00312}{H\+F\+::\+Geometry}} is \mbox{\hyperlink{a01160}{Mesh\+Info}}, which stores the vertices and indices of meshes as matricies in \mbox{\hyperlink{a00313}{Eigen}}.

\begin{DoxyRemark}{Remarks}
Right now only O\+BJ files are supported for loading geometry from disk, however \mbox{\hyperlink{a01160}{Mesh\+Info}} offers multiple constructors that accept arrays of vertices and indices so support for more formats can be developed.
\end{DoxyRemark}
\begin{DoxySeeAlso}{See also}
\mbox{\hyperlink{a00312_ad32d656c7a8c84beded49e1f14f64176}{Load\+Mesh\+Objects}} to load O\+BJ files from disk. 

\mbox{\hyperlink{a01160}{Mesh\+Info}} for details on how Human\+Factors represents Mesh geometry. 
\end{DoxySeeAlso}

If the x,y,z location in the array hasn\textquotesingle{}t been seen yet, add its coordinates to mapped\+\_\+vertices, assign it a unique ID in a an index hashmap, then add its id to mapped\+\_\+indexes. \begin{DoxyPrecond}{Precondition}
