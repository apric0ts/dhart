\doxysubsection{Detailed Description}
A graph representing connections between points in space. 

Every \mbox{\hyperlink{struct_human_factors_1_1_spatial_structures_1_1_node}{Node}} in the graph contains an X,Y,Z coordinate which can be used to represent a specific point in space. This graph internally is stored as a Sparse Matrix for space efficency. Nodes are stored in a hashmap containing X,Y, and Z coordinates, allowing for quick indexing of specific nodes by location alone. Access to this graph\textquotesingle{}s internal C\+SR is available through \mbox{\hyperlink{class_human_factors_1_1_spatial_structures_1_1_graph_a84a7286932920179d90a63cfd8ada207}{Graph.\+Compress\+To\+C\+S\+R()}}.

\begin{DoxyParagraph}{Cost Types}
This \mbox{\hyperlink{class_human_factors_1_1_spatial_structures_1_1_graph}{Graph}} is capable of holding multiple cost types for any of it\textquotesingle{}s edges. Each cost type has a distinct key as it\textquotesingle{}s name, such as \char`\"{}\+Cross\+Slope\char`\"{} or \char`\"{}\+Energy\+Expenditure\char`\"{}. Upon creation, the graph is assigned a default cost type, {\ttfamily Distance} which can be accessed explicitly by the key \char`\"{}\+Distance\char`\"{} or leaving the cost\+\_\+type field blank. Alternate costs have corresponding edges in the default cost set, but different costs to traverse from the parent to the child node.
\end{DoxyParagraph}
\begin{DoxyParagraph}{Node\+Attributes}
The graph is able to store an arbitrary amount of information about the nodes it contains as strings. Similar to alternate cost types, node attributes are each have a distinct key as their name, but instead of conatining information about edges in the graph, node attributes contain information about nodes. Unlike the cost algorithms in edgecosts, right now there is no functionality within \mbox{\hyperlink{namespace_human_factors}{Human\+Factors}} that populates the node attributes of the graph with any kind of metric, however the methods to add and clear node attributes are made available so you are free to add your own node attributes.
\end{DoxyParagraph}
\begin{DoxyInvariant}{Invariant}
1) The C\+SR maintained by this graph will always be valid. 

2) Every unique unique node in the graph will be assigned a unique id. A \mbox{\hyperlink{struct_human_factors_1_1_spatial_structures_1_1_node}{Node}} is considered unique if it has an X,Y,Z location that is not within 0.\+0001 units of any other node in the graph.
\end{DoxyInvariant}
\begin{DoxyNote}{Note}
The graph offers some basic functionality to add edges and nodes but it\textquotesingle{}s main use is to provide access to the output of the \mbox{\hyperlink{namespace_human_factors_1_1_graph_generator}{Graph\+Generator}} and \mbox{\hyperlink{namespace_human_factors_1_1_visibility_graph}{Visibility\+Graph}}. If adding edges or alternate cost types please make sure to read the documentation for these functions and that all preconditions are followed.
\end{DoxyNote}
